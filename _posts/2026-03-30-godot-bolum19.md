# Squash the Creeps! — Bölüm 3: Oyuncuyu Kodla

Sahne ve input action'lar hazır. Artık kod yazma zamanı! Bu bölümde karakterin hareketi için script yazacak, ardından kamera ekleyerek sahneyi test edeceğiz.

---

## Scripti Oluştur

**Player** node'una sağ tıkla ve **"Attach Script"** seçeneğini seç:

![Scripti Ekle](/assets/images/01.attach_script_to_player.webp)
*Player node'una sağ tıklayıp "Attach Script" ile script ekliyoruz*

Script oluşturma penceresinde **Template** değerini **"Empty"** olarak ayarla ve **Create** tıkla. Oyuncu hareketini sıfırdan kendimiz yazacağız.

---

## Değişkenleri Tanımla

Scriptin başına şu değişkenleri ekle:

```gdscript
extends CharacterBody3D

# Oyuncunun metre/saniye cinsinden hareket hızı
@export var speed = 14

# Havadayken aşağı doğru ivme (yerçekimi), metre/saniye kare
@export var fall_acceleration = 75

var target_velocity = Vector3.ZERO
```

- **`speed`**: Karakterin zemin üzerindeki hareket hızı
- **`fall_acceleration`**: Havadayken uygulanacak yerçekimi ivmesi
- **`target_velocity`**: Karakterin hedef hız vektörü — her karede güncellenip yeniden kullanılır

> 💡 **2D'den farkı:** 3D'de birimler metre cinsinden. 2D'de bin piksel ekranın yarısına karşılık gelirken, 3D'de bu bir kilometre eder! Bu yüzden değerler çok farklı.

---

## `_physics_process()` ile Hareketi Kodla

3D'de fizikle ilgili tüm hesaplamalar `_physics_process()` içinde yapılır. Bu fonksiyon sabit zaman aralıklarıyla çalışır ve kinematik/rigid body hareketi için özel olarak tasarlanmıştır.

### 1. Girdi Yönünü Hesapla

```gdscript
func _physics_process(delta):
    var direction = Vector3.ZERO

    if Input.is_action_pressed("move_right"):
        direction.x += 1
    if Input.is_action_pressed("move_left"):
        direction.x -= 1
    if Input.is_action_pressed("move_back"):
        direction.z += 1
    if Input.is_action_pressed("move_forward"):
        direction.z -= 1
```

> 💡 **3D'de zemin düzlemi XZ'dir.** Y ekseni yukarı/aşağı yönünü temsil eder. İleri-geri hareket Z ekseninde, sağ-sol hareket X ekseninde olur.

Bu dört koşul, 8 farklı yön kombinasyonu üretir. W+D aynı anda basılırsa vektörün uzunluğu yaklaşık 1.4 olur — diagonal hareket daha hızlı olurdu. Bunu engellemek için `normalized()` kullanıyoruz.

### 2. Pivotu Yöne Döndür

```gdscript
    if direction != Vector3.ZERO:
        direction = direction.normalized()
        # Basis özelliğini ayarlamak node'un rotasyonunu etkiler
        $Pivot.basis = Basis.looking_at(direction)
```

- Yön vektörü sıfır değilse (yani tuşa basılıyorsa) normalize ediyoruz
- `Basis.looking_at(direction)` ile `Pivot` node'unu hareket yönüne döndürüyoruz
- Böylece karakter modeli baktığı yöne dönüyor

### 3. Hız Hesapla ve Karakteri Hareket Ettir

```gdscript
    # Yatay hız
    target_velocity.x = direction.x * speed
    target_velocity.z = direction.z * speed

    # Dikey hız (yerçekimi)
    if not is_on_floor():
        target_velocity.y = target_velocity.y - (fall_acceleration * delta)

    # Karakteri hareket ettir
    velocity = target_velocity
    move_and_slide()
```

**Yatay hız:** `direction * speed` ile karakterin XZ düzlemindeki hızını belirliyoruz.

**Dikey hız (yerçekimi):** `is_on_floor()` her karede zemine temas olup olmadığını kontrol eder. Karakter havadaysa `fall_acceleration * delta` kadar aşağı çekiyoruz. Bu, gerçekçi bir yerçekimi simülasyonu sağlar.

**`move_and_slide()`:** `CharacterBody3D`'nin en güçlü metodlarından biri — karakteri yumuşak biçimde hareket ettirir. Duvara çarparsa motor hareketi otomatik olarak düzleştirir. Dahili `velocity` değişkenini kullanır.

## Tam Script

```gdscript
extends CharacterBody3D

@export var speed = 14
@export var fall_acceleration = 75

var target_velocity = Vector3.ZERO

func _physics_process(delta):
    var direction = Vector3.ZERO

    if Input.is_action_pressed("move_right"):
        direction.x += 1
    if Input.is_action_pressed("move_left"):
        direction.x -= 1
    if Input.is_action_pressed("move_back"):
        direction.z += 1
    if Input.is_action_pressed("move_forward"):
        direction.z -= 1

    if direction != Vector3.ZERO:
        direction = direction.normalized()
        $Pivot.basis = Basis.looking_at(direction)

    target_velocity.x = direction.x * speed
    target_velocity.z = direction.z * speed

    if not is_on_floor():
        target_velocity.y = target_velocity.y - (fall_acceleration * delta)

    velocity = target_velocity
    move_and_slide()
```

## Ortografik Kamereya Geç

Bu oyun için perspektif yerine **ortografik (orthogonal) projeksiyon** daha uygun — oyun alanını daha iyi çerçeveler ve mesafeleri okumayı kolaylaştırır.

Camera3D'yi seç ve Inspector'da şu değerleri ayarla:

![Camera3D Değerleri](/assets/images/13.camera3d_values.webp)
*Projection: Orthogonal, Size: 19*

- **Projection:** `Orthogonal`
- **Size:** `19`

Karakter artık daha "yassı" görünür ve zemin arka planı doldurur.

> 💡 **Gölge kalitesi notu:** Ortografik kamera kullanırken gölge kalitesi kameranın **Far** değerine bağlıdır. Gölgeler bulanık görünüyorsa Far değerini `100` gibi daha düşük bir değere çek — ama çok düşürme, uzaktaki nesneler kaybolmaya başlar.

## Özet

| Kavram | Açıklama |
|---|---|
| `_physics_process()` | Fizik hesaplamaları için sabit zamanlı döngü |
| `Vector3.ZERO` | Uzunluğu sıfır olan 3D vektör — hareketsizlik |
| `direction.normalized()` | Diagonal hareketi düzeltmek için vektörü normalize et |
| `Basis.looking_at()` | Node'u belirli bir yöne döndürür |
| `is_on_floor()` | Zemine temas olup olmadığını kontrol eder |
| `move_and_slide()` | CharacterBody3D'yi yumuşak biçimde hareket ettirir |
| `Orthogonal` projeksiyon | Perspektif bozulması olmadan oyun alanını görüntüler |
